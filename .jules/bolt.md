## 2024-05-24 - Avoiding String cloning and Vec allocations in DAG traversal hot paths
**Learning:** In the core scenario execution loop (`run_one_pass` and related `step_statuses` updates), the keys (Step IDs) are already known and present in the `step_statuses` HashMap (initialized once upfront). Updating statuses by doing `self.step_statuses.insert(step_id.clone(), ...)` inside the loop incurs unnecessary String allocation and cloning. Similarly, iterating over `.scenario.get_steps()` which returns a new `Vec` incurs an allocation.
**Action:** When updating existing entries in a `HashMap`, use `get_mut(key)` instead of `insert(key.clone(), value)` if the key string is expensive to clone. When iterating over a map just to check values, use `.values()` instead of allocating a `Vec` of items.
